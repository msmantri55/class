3. Voting

Due: Sun, 21 Feb, 11:59pm, Java and Python, Isaac, cs373pj3.

Specification

Write a program to solve the following UVa problem: Australian Voting.

When reassigning the ballots of a loser, you must not iterate over all the ballots.

Analysis

These are additional descriptions of the underlying math:

  • Arrow’s Theorem
  • Preferential Voting
  • Range Voting

Issues

Add the following to Google Project’s issue tracker:

Phase Description
1 design Sketch out an algorithm. Write it on a Wiki page.
2 Sketch it out again with any optimizations. Write it on a Wiki page.
3 Sketch out all the interfaces.
4 source control Use a source control system (e.g. Subversion, Git, Mercurial) to manage your files.
5 testing Create a set of unit tests for each of the functions above. Use Blackboard to reach consensus on the unit tests.
6 Create a set of acceptance tests in Voting.in / Voting.out. Again, use Blackboard to reach consensus on the acceptance tests.
7 C++ only: Use Valgrind for all tests.
8 implentation Use assert to check pre-conditions, post-conditions, argument validity, return-value validity, and invariants.
9 C++ only: Do not use new, delete, malloc() or free(). This problem is best solved using only the stack.
10 Worry about this last, but your program should run as fast as possible and use as little memory as possible.
11 bug tracking Use Google Project’s issues to track your bugs.
12 documentation C++ only: Use Doxygen to document the interfaces.
Java only: Use javdoc to document the interfaces.
Python only: Use pydoc to document the interfaces.
13 Use Google Project’s wiki to document the implementation.
14 Use comments only if you need to explain the why of a particular implementation.
15 Choose a coding convention, and be consistent. Use good variable names. Write readable code with good indentation, blank lines, and blank spaces.
16 overview Fill out the overview.
17 submission Submit the content below.
18 postings Post the content below.
