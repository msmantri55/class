5. Faculty1

Due: Sun, 21 Mar, 11:59pm, Python, Isaac, cs373pj5.

Specification

Design a system to manage faculty information pertaining to teaching and research.

The system will consist of the following for each faculty member:

  • name
  • office
  • phone
  • e-mail
  • website
  • office hours
  • type (e.g. lecturer, professor, researcher)
  • degrees earned: type, institution, and date
  • research area: name
  • graduate students: name, type (Master’s, Ph.D.), theses/dissertation, and date
  • courses: name, number, type (undergrad, grad), class unique, and class semester
  • journals: name, date, and article title
  • conferences: name, date, location, and presentation title
  • publishers: name, book title, book co-authors (UT CS only), and book date
  • awards: name, type, and date

Design a database schema and decide on the following:

  • the names of the tables and attributes
  • the primary key of each table
  • the foreign keys of each table

Normalize the design. Eliminate multi-valued attributes, repeated attributes, and subkeys.

Create a UML diagram to represent your design. Use any UML editor that you like. The diagram must clearly show the associations and multiplicity between the tables.

Collaboratively design an XML Schema for your data with the other groups.

Use the department’s website to obtain information about ten of each of the following:

  • faculty
  • research areas
  • graduate students
  • courses
  • journals
  • conferences
  • publishers
  • awards
